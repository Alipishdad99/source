جلسه ۰۹: نشست‌ها (Sessions) و امنیت ورود

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: پیاده‌سازی لاگین و خروج، محافظت از صفحات با Session

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

در این جلسه، دانشجویان با مفهوم نشست (Session) آشنا می‌شوند و یاد می‌گیرند که چگونه یک سیستم لاگین و خروج امن پیاده‌سازی کنند.

  1. مفهوم Session و Cookie (۲۰ دقیقه): تفاوت، کاربرد، نحوه ذخیره‌سازی و امنیت.
  2. شروع و مدیریت Session (۳۰ دقیقه): session_start()، ذخیره و خواندن متغیرهای جلسه.
  3. پیاده‌سازی لاگین و خروج (۳۰ دقیقه): احراز هویت با دیتابیس و مدیریت جلسه.
  4. تمرین عملی: محافظت از admin.php (۴۰ دقیقه): فقط کاربران لاگین شده دسترسی داشته باشند.
📌 نکته مهم برای مدرس:
  • این جلسه، امنیت برنامه را کامل می‌کند.
  • حتماً تفاوت Session و Cookie را به‌خوبی توضیح دهید.
  • تأکید کنید که رمز عبور هرگز نباید در Session ذخیره شود.
  • از header() برای هدایت کاربران لاگین نشده استفاده کنید.

بخش دوم: مفهوم Session و Cookie (۲۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان، هر بار که کاربر صفحه‌ای را باز می‌کرد، سرور او را فراموش می‌کرد. چون HTTP یک پروتکل بی‌حالت (Stateless) است. یعنی اگر کاربر لاگین کند و سپس به صفحه دیگری برود، سرور نمی‌داند که او قبلاً لاگین کرده است! برای حل این مشکل، از نشست (Session) استفاده می‌کنیم. Session یک فضای موقت روی سرور است که اطلاعات کاربر را در طول بازدیدهای مختلف نگهداری می‌کند. تفاوت Session و Cookie: - Cookie روی مرورگر کاربر ذخیره می‌شود (سمت کلاینت). - Session روی سرور ذخیره می‌شود (سمت سرور) و فقط یک شناسه (Session ID) به کاربر داده می‌شود.»

۱. مقایسه Session و Cookie:

ویژگی Session Cookie
محل ذخیره سرور مرورگر کاربر
امنیت ✅ بالا ❌ پایین
حجم داده ✅ بدون محدودیت ❌ حداکثر ۴ کیلوبایت
مدت اعتبار تا بسته شدن مرورگر قابل تنظیم
کاربرد اصلی لاگین، سبد خرید تنظیمات، ردیابی

۲. نحوه کار Session:

🌐 مرورگر → 🖥️ سرور (session_start()) → 💾 فایل Session

بخش سوم: شروع و مدیریت Session (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «برای کار با Session، باید: ۱. جلسه را با session_start() شروع کنیم. ۲. داده‌ها را در $_SESSION ذخیره کنیم. ۳. داده‌ها را از $_SESSION بخوانیم. ۴. در زمان خروج، جلسه را با session_destroy() پایان دهیم. session_start() باید در بالای تمام صفحاتی که از Session استفاده می‌کنند، قرار گیرد.»

۱. شروع و ذخیره در Session:

<?php
// ===== شروع جلسه =====
session_start();

// ===== ذخیره داده در جلسه =====
$_SESSION['user_id'] = 1;
$_SESSION['username'] = 'ali_developer';

// ===== خواندن داده از جلسه =====
if (isset($_SESSION['username'])) {
    echo "خوش آمدید " . $_SESSION['username'];
}

// ===== حذف یک متغیر =====
unset($_SESSION['user_id']);

// ===== پایان کامل جلسه =====
session_destroy();
?>

۲. توابع مدیریت Session با MySQLi:

<?php
// ===== فایل: functions/session.php (MySQLi) =====

function startSession() {
    if (session_status() == PHP_SESSION_NONE) {
        session_start();
    }
}

function isLoggedIn() {
    startSession();
    return isset($_SESSION['user_id']) && isset($_SESSION['username']);
}

function getCurrentUsername() {
    startSession();
    return isset($_SESSION['username']) ? $_SESSION['username'] : null;
}

function getCurrentUserId() {
    startSession();
    return isset($_SESSION['user_id']) ? $_SESSION['user_id'] : null;
}

function loginUser($user_id, $username) {
    startSession();
    session_regenerate_id(true);
    $_SESSION['user_id'] = $user_id;
    $_SESSION['username'] = $username;
}

function logoutUser() {
    startSession();
    $_SESSION = [];
    session_destroy();
}

function protectPage() {
    startSession();
    if (!isLoggedIn()) {
        header('Location: login.php');
        exit;
    }
}
?>

۲. فایل login_process.php (پردازش لاگین با MySQLi):

<?php
// ===== فایل: login_process.php (MySQLi) =====
require_once 'config/db.php';
require_once 'functions/session.php';

if ($_SERVER["REQUEST_METHOD"] != "POST") {
    header('Location: login.php');
    exit;
}

$username = isset($_POST['username']) ? trim($_POST['username']) : '';
$password = isset($_POST['password']) ? $_POST['password'] : '';

if (empty($username) || empty($password)) {
    session_start();
    $_SESSION['login_error'] = 'نام کاربری و رمز عبور الزامی است.';
    header('Location: login.php');
    exit;
}

$conn = getDBConnection();

// بررسی کاربر در دیتابیس
$stmt = $conn->prepare("SELECT id, username, password, is_active FROM users WHERE username = ?");
$stmt->bind_param("s", $username);
$stmt->execute();
$result = $stmt->get_result();
$user = $result->fetch_assoc();
$stmt->close();

if (!$user || $user['password'] != $password) {
    $conn->close();
    session_start();
    $_SESSION['login_error'] = 'نام کاربری یا رمز عبور اشتباه است.';
    header('Location: login.php');
    exit;
}

if ($user['is_active'] != 1) {
    $conn->close();
    session_start();
    $_SESSION['login_error'] = 'حساب کاربری شما غیرفعال است.';
    header('Location: login.php');
    exit;
}

$conn->close();

// لاگین موفق
loginUser($user['id'], $user['username']);
header('Location: admin.php');
exit;
?>
