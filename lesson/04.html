جلسه ۰۴: توابع و مدیریت ورودی‌های فرم

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: دریافت داده از فرم، اعتبارسنجی با توابع و نمایش خطاهای داینامیک

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

این جلسه یکی از مهم‌ترین جلسات دوره است. دانشجویان یاد می‌گیرند چگونه داده‌های کاربر را دریافت کنند، آن‌ها را اعتبارسنجی کنند و با استفاده از توابع، کدهای خود را سازماندهی کنند.

  1. مرور فرم‌ها و متدهای GET/POST (۲۰ دقیقه): مرور فرم جلسه اول، تفاوت GET و POST، کاربرد هر کدام.
  2. متغیرهای سوپرگلوبال $_GET و $_POST (۳۰ دقیقه): نحوه دریافت داده از فرم، بررسی وجود داده، نمایش داده‌ها.
  3. تعریف و استفاده از توابع (۳۰ دقیقه): ساختار توابع، آرگومان‌ها، مقدار بازگشتی، دامنه متغیرها.
  4. تمرین عملی: اعتبارسنجی فرم ثبت‌نام (۴۰ دقیقه): دریافت داده، اعتبارسنجی با توابع، نمایش پیام‌های خطا و موفقیت.
📌 نکته مهم برای مدرس: در این جلسه، دانشجویان با دو مفهوم کلیدی آشنا می‌شوند:
  • ارتباط با کاربر: چگونه داده‌های فرم را دریافت کنیم.
  • سازماندهی کد: چگونه با توابع، کدهای خود را تمیز و قابل استفاده مجدد کنیم.
تأکید کنید که اعتبارسنجی (Validation) یکی از مهم‌ترین مباحث امنیتی است. هیچ‌گاه به داده‌های ورودی کاربر اعتماد نکنید!

بخش دوم: متغیرهای سوپرگلوبال $_GET و $_POST (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان فرم‌هایی ساختیم که فقط ظاهر داشتند و هیچ کاری نمی‌کردند. امروز قرار است به این فرم‌ها جان بدهیم! وقتی کاربر روی دکمه ارسال کلیک می‌کند، داده‌های فرم به سرور فرستاده می‌شوند. در PHP، این داده‌ها را با استفاده از متغیرهای سوپرگلوبال دریافت می‌کنیم. دو متغیر سوپرگلوبال مهم برای دریافت داده‌های فرم: ۱. $_GET → برای داده‌هایی که با متد GET ارسال می‌شوند. ۲. $_POST → برای داده‌هایی که با متد POST ارسال می‌شوند. این متغیرها در همه جای برنامه در دسترس هستند (سوپرگلوبال یعنی همه‌جا قابل دسترس).»

۱. تفاوت GET و POST:

ویژگی GET POST
مکان داده در URL (نوار آدرس) در بدنه درخواست (HTTP Body)
قابلیت مشاهده قابل مشاهده برای همه پنهان (در URL دیده نمی‌شود)
محدودیت حجم حداکثر ۲۰۴۸ کاراکتر بدون محدودیت (تا چند مگابایت)
قابل نشانه‌گذاری (Bookmark) بله خیر
ایمنی برای رمز عبور ❌ خطرناک ✅ امن
کاربرد اصلی جستجو، فیلتر، صفحه‌بندی ثبت‌نام، لاگین، فرم‌های طولانی
📌 قانون طلایی:
  • برای نمایش داده (مثل جستجو و فیلتر) از GET استفاده کنید.
  • برای تغییر داده (مثل ثبت‌نام، لاگین، حذف) از POST استفاده کنید.
  • هرگز اطلاعات حساس (رمز عبور، شماره کارت) را با GET ارسال نکنید!

۲. دریافت داده با $_GET:

<?php
// ===== مثال: دریافت از URL =====
// آدرس: http://localhost/lesson/profile.php?name=ali&age=25

// دریافت داده از URL
$name = $_GET['name'];  // 'ali'
$age = $_GET['age'];    // '25'

echo "نام: " . $name . "<br>";
echo "سن: " . $age . "<br>";

// ===== بررسی وجود پارامتر =====
if (isset($_GET['id'])) {
    $user_id = $_GET['id'];
    echo "شناسه کاربر: " . $user_id;
} else {
    echo "شناسه کاربر مشخص نشده است.";
}

// ===== نمایش تمام پارامترهای GET =====
print_r($_GET);

// ===== ساخت لینک با پارامترهای GET =====
echo '<a href="page.php?page=2&sort=name">صفحه ۲ با مرتب‌سازی نام</a>';
?>

۳. دریافت داده با $_POST:

<?php
// ===== مثال: دریافت از فرم =====
// فرم با method="POST"

if ($_SERVER["REQUEST_METHOD"] == "POST") {
    // دریافت داده از فرم
    $username = $_POST['username'];
    $email = $_POST['email'];
    $password = $_POST['password'];

    echo "نام کاربری: " . $username . "<br>";
    echo "ایمیل: " . $email . "<br>";
    echo "رمز عبور: " . $password . "<br>";
}

// ===== بررسی وجود داده =====
if (isset($_POST['submit'])) {
    // کاربر دکمه ارسال را زده است
    $data = $_POST;
    // پردازش داده‌ها
}

// ===== نمایش تمام داده‌های POST =====
print_r($_POST);

// ===== $_REQUEST =====
// $_REQUEST شامل داده‌های GET، POST و COOKIE است
// اما استفاده از آن توصیه نمی‌شود (به دلیل عدم شفافیت)
$data = $_REQUEST['name'];
?>

بخش سوم: توابع در PHP (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان کدهایمان را خط به خط نوشته‌ایم. اما در پروژه‌های واقعی، کدها هزاران خط می‌شوند. اگر همه کدها را بدون سازماندهی بنویسیم، کار با آن‌ها غیرممکن می‌شود. اینجا است که توابع به کمک ما می‌آیند. تابع یک بلوک کد است که یک کار خاص را انجام می‌دهد. می‌توانیم یک تابع را یک بار بنویسیم و بارها و بارها از آن استفاده کنیم. به تابع مثل یک ماشین لباسشویی فکر کنید! شما لباس‌ها را داخل آن می‌ریزید (ورودی)، ماشین کار شستشو را انجام می‌دهد (پردازش)، و لباس‌های تمیز را تحویل می‌دهد (خروجی).»

۱. ساختار توابع:

ورودی (آرگومان‌ها)
→
پردازش (بدنه تابع)
→
خروجی (return)
<?php
// ===== ساختار پایه تابع =====
function function_name($parameter1, $parameter2) {
    // بدنه تابع
    // ...
    return $result;
}

// ===== مثال ۱: تابع ساده بدون آرگومان =====
function sayHello() {
    echo "سلام دنیا!<br>";
}

sayHello(); // فراخوانی: سلام دنیا!

// ===== مثال ۲: تابع با آرگومان =====
function greetUser($name) {
    echo "سلام " . $name . "! خوش آمدید.<br>";
}

greetUser("علی"); // سلام علی! خوش آمدید.
greetUser("سارا"); // سلام سارا! خوش آمدید.

// ===== مثال ۳: تابع با مقدار بازگشتی =====
function add($a, $b) {
    return $a + $b;
}

$result = add(5, 3);
echo "۵ + ۳ = " . $result . "<br>"; // ۵ + ۳ = 8

// ===== مثال ۴: تابع با آرگومان پیش‌فرض =====
function power($base, $exponent = 2) {
    return pow($base, $exponent);
}

echo power(3) . "<br>";   // 9 (3^2)
echo power(3, 3) . "<br>"; // 27 (3^3)

// ===== مثال ۵: تابع با چندین مقدار بازگشتی (آرایه) =====
function getUserInfo($id) {
    // در واقعیت، از دیتابیس می‌خوانیم
    return [
        "id" => $id,
        "name" => "کاربر " . $id,
        "email" => "user" . $id . "@site.com"
    ];
}

$user = getUserInfo(5);
echo "نام: " . $user['name'] . "<br>";
echo "ایمیل: " . $user['email'] . "<br>";
?>

۲. دامنه متغیرها (Variable Scope):

<?php
// ===== متغیر سراسری (Global) =====
$global_var = "من یک متغیر سراسری هستم";

function testScope() {
    // ===== متغیر محلی (Local) =====
    $local_var = "من یک متغیر محلی هستم";

    echo "داخل تابع: " . $local_var . "<br>";

    // برای دسترسی به متغیر سراسری باید از کلیدواژه global استفاده کرد
    global $global_var;
    echo "داخل تابع (global): " . $global_var . "<br>";

    // یا از $GLOBALS استفاده کنید
    echo "داخل تابع (GLOBALS): " . $GLOBALS['global_var'] . "<br>";
}

testScope();

// متغیر محلی خارج از تابع قابل دسترس نیست
// echo $local_var; // خطا: undefined variable

// متغیر سراسری خارج از تابع قابل دسترس است
echo "خارج از تابع: " . $global_var . "<br>";

// ===== متغیرهای استاتیک (Static) =====
function counter() {
    static $count = 0;
    $count++;
    echo "تعداد فراخوانی: " . $count . "<br>";
}

counter(); // 1
counter(); // 2
counter(); // 3
// متغیر static مقدار خود را بین فراخوانی‌ها حفظ می‌کند
?>

۳. توابع مفید برای اعتبارسنجی:

<?php
// ===== توابع اعتبارسنجی =====

// ۱. بررسی خالی بودن
function isEmpty($value) {
    return empty(trim($value));
}

// ۲. بررسی طول رشته
function isLengthValid($value, $min, $max) {
    $length = strlen(trim($value));
    return $length >= $min && $length <= $max;
}

// ۳. بررسی ایمیل
function isValidEmail($email) {
    return filter_var($email, FILTER_VALIDATE_EMAIL) !== false;
}

// ۴. بررسی شماره تلفن (ساده)
function isValidPhone($phone) {
    return preg_match('/^[0-9]{11}$/', $phone) === 1;
}

// ۵. بررسی عدد بودن
function isNumeric($value) {
    return is_numeric($value);
}

// ۶. بررسی دامنه سن
function isValidAge($age, $min = 18, $max = 100) {
    return is_numeric($age) && $age >= $min && $age <= $max;
}

// ۷. پاک‌سازی ورودی (برای جلوگیری از XSS)
function sanitize($value) {
    return htmlspecialchars(trim($value), ENT_QUOTES, 'UTF-8');
}

// ===== استفاده از توابع =====
$username = "  علی  ";
$email = "ali@site.com";
$phone = "09123456789";
$age = 25;

if (isEmpty($username)) {
    echo "❌ نام کاربری نمی‌تواند خالی باشد.<br>";
} else {
    echo "✅ نام کاربری: " . sanitize($username) . "<br>";
}

if (isValidEmail($email)) {
    echo "✅ ایمیل معتبر است.<br>";
} else {
    echo "❌ ایمیل نامعتبر است.<br>";
}

if (isValidPhone($phone)) {
    echo "✅ شماره تلفن معتبر است.<br>";
} else {
    echo "❌ شماره تلفن نامعتبر است.<br>";
}

if (isValidAge($age)) {
    echo "✅ سن معتبر است.<br>";
} else {
    echo "❌ سن نامعتبر است.<br>";
}
?>
📌 نکته مهم:
  • از htmlspecialchars() برای جلوگیری از حملات XSS (Cross-Site Scripting) استفاده کنید.
  • از filter_var() با FILTER_VALIDATE_EMAIL برای اعتبارسنجی ایمیل استفاده کنید.
  • از preg_match() برای اعتبارسنجی با الگوی Regex استفاده کنید.
  • هیچ‌گاه به ورودی کاربر اعتماد نکنید! همیشه آن را اعتبارسنجی و پاک‌سازی کنید.

  4. سناریوهای مختلف را تست کنید:

    • ایمیل نامعتبر وارد کنید → خطای ایمیل نمایش داده شود.
    • رمز عبور با تأیید مطابقت نداشته باشد → خطا نمایش داده شود.
    • همه فیلدها را درست پر کنید → پیام موفقیت نمایش داده شود.
  5. مقادیر فیلدها بعد از ارسال، در فرم باقی می‌مانند (بررسی کنید).
🧪 چالش‌های اضافی برای دانشجویان قوی‌تر:

چالش ۱: اعتبارسنجی در سمت کلاینت

با استفاده از HTML5 (مثل required، pattern، min و max) اعتبارسنجی سمت کلاینت را اضافه کنید.

چالش ۲: کپچا (CAPTCHA) ساده

یک سوال ساده ریاضی (مثل ۵ + ۳ = ؟) به فرم اضافه کنید و پاسخ را اعتبارسنجی کنید.

چالش ۳: ذخیره در فایل

به جای نمایش پیام موفقیت، داده‌های ثبت‌نام را در یک فایل users.txt ذخیره کنید.

راهنما: از file_put_contents() با FILE_APPEND استفاده کنید.

چالش ۴: فرم چند مرحله‌ای

فرم را به دو مرحله تقسیم کنید: مرحله اول (نام و ایمیل) و مرحله دوم (رمز عبور و تلفن).
