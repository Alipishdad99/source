جلسه ۰۶: اتصال PHP به دیتابیس با MySQLi

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: اتصال به دیتابیس با MySQLi، استفاده از Prepared Statements و جلوگیری از SQL Injection

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

این جلسه، پلی است بین دنیای PHP و دنیای دیتابیس. دانشجویان یاد می‌گیرند که چگونه به‌صورت امن و حرفه‌ای به دیتابیس متصل شوند. امنیت مهم‌ترین موضوع این جلسه است.

  1. معرفی MySQLi و دو روش اتصال (۲۰ دقیقه): MySQLi چیست؟ تفاوت روش رویه‌ای و شیءگرا، مزایا و معایب.
  2. ایجاد اتصال با روش رویه‌ای و شیءگرا (۳۰ دقیقه): نحوه اتصال، مدیریت خطاها، انتخاب دیتابیس، تنظیم charset.
  3. Prepared Statements با MySQLi (۳۰ دقیقه): مفهوم، نحوه استفاده، مزایا در مقابل کوئری‌های مستقیم.
  4. تمرین عملی: ذخیره فرم در دیتابیس (۴۰ دقیقه): اتصال به دیتابیس و ذخیره پیام‌های Guestbook با MySQLi.
📌 نکته مهم برای مدرس:
  • این جلسه یکی از مهم‌ترین جلسات دوره از نظر امنیت است.
  • حتماً مفهوم SQL Injection را با مثال عملی نشان دهید.
  • تأکید کنید که همیشه از Prepared Statements استفاده کنند.
  • هر دو روش رویه‌ای (Procedural) و شیءگرا (Object-Oriented) را آموزش دهید.
  • تفاوت mysqli_query() و mysqli_prepare() را توضیح دهید.

بخش دوم: معرفی MySQLi (۲۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان با phpMyAdmin کار کردیم و کوئری‌های SQL را دستی اجرا کردیم. اما در یک وب‌سایت واقعی، کاربران نمی‌توانند مستقیم به phpMyAdmin بروند! باید از طریق کد PHP به دیتابیس متصل شویم و کوئری‌ها را اجرا کنیم. PHP دو روش اصلی برای اتصال به دیتابیس MySQL دارد: ۱. MySQLi (MySQL Improved - مخصوص MySQL) ۲. PDO (PHP Data Objects - عمومی) ما در این جلسه از MySQLi استفاده می‌کنیم چون: - مخصوص MySQL است و بهینه‌تر کار می‌کند. - هم به‌صورت رویه‌ای و هم شیءگرا قابل استفاده است. - از Prepared Statements پشتیبانی کامل می‌کند. - در اکثر هاست‌ها به‌صورت پیش‌فرض فعال است. MySQLi دو روش دارد: - رویه‌ای (Procedural): با توابع مثل mysqli_connect() - شیءگرا (Object-Oriented): با کلاس mysqli»

۱. MySQLi چیست؟

  • MySQLi مخفف MySQL Improved است.
  • نسخه بهبودیافته و مدرن‌تر از پسوند قدیمی MySQL است.
  • از Prepared Statements پشتیبانی کامل می‌کند.
  • هم به‌صورت رویه‌ای و هم شیءگرا قابل استفاده است.
  • برای پروژه‌هایی که فقط از MySQL استفاده می‌کنند، انتخاب مناسبی است.

۲. مقایسه دو روش MySQLi:

📌 روش رویه‌ای (Procedural)

  • استفاده از توابع مثل mysqli_connect()
  • نیاز به پاس دادن $connection به هر تابع
  • ساده‌تر برای مبتدیان
  • مثال: mysqli_query($conn, $sql)

📌 روش شیءگرا (Object-Oriented)

  • استفاده از کلاس mysqli
  • کد تمیزتر و سازماندهی‌شده‌تر
  • مناسب برای پروژه‌های بزرگ
  • مثال: $conn->query($sql)
📌 توصیه: در این جلسه، هر دو روش را یاد می‌گیریم اما تأکید اصلی روی روش شیءگرا است چون با مفاهیم OOP که در جلسه ۱۰ یاد می‌گیرید، هماهنگ‌تر است.

۳. نمایش SQL Injection (حمله تزریق SQL):

🚨 حمله SQL Injection چیست؟

وقتی ورودی کاربر مستقیماً در کوئری SQL قرار می‌گیرد، هکر می‌تواند با وارد کردن کد SQL مخرب، به دیتابیس نفوذ کند.

مثال خطرناک:

// کد PHP خطرناک
$username = $_POST['username'];  // کاربر می‌نویسد: ' OR '1'='1
$query = "SELECT * FROM users WHERE username = '$username'";

// کوئری نهایی:
SELECT * FROM users WHERE username = '' OR '1'='1'
// این کوئری تمام کاربران را برمی‌گرداند!

✅ راه حل: استفاده از Prepared Statements

با Prepared Statements، داده و کوئری از هم جدا می‌شوند و ورودی کاربر به‌عنوان داده تفسیر می‌شود، نه کد SQL.

بخش سوم: ایجاد اتصال با MySQLi (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «برای اتصال به دیتابیس با MySQLi، به اطلاعات زیر نیاز داریم: - Host: آدرس سرور (معمولاً localhost) - Username: نام کاربری (در XAMPP: root) - Password: رمز عبور (در XAMPP: خالی) - Database name: نام دیتابیس اتصال به دیتابیس ممکن است با خطا مواجه شود. برای مدیریت خطاها، از شرط‌ها و try/catch استفاده می‌کنیم. دو روش برای اتصال داریم: رویه‌ای و شیءگرا.»

۱. روش رویه‌ای (Procedural):

<?php
// ===== اتصال به دیتابیس با روش رویه‌ای =====
$host = 'localhost';
$username = 'root';
$password = '';
$dbname = 'guestbook_db';

// ایجاد اتصال
$conn = mysqli_connect($host, $username, $password, $dbname);

// بررسی اتصال
if (!$conn) {
    die("❌ خطا در اتصال: " . mysqli_connect_error());
}

// تنظیم charset برای پشتیبانی از فارسی
mysqli_set_charset($conn, "utf8mb4");

echo "✅ اتصال با موفقیت برقرار شد.";

// بستن اتصال
mysqli_close($conn);
?>

۲. روش شیءگرا (Object-Oriented):

<?php
// ===== اتصال به دیتابیس با روش شیءگرا =====
$host = 'localhost';
$username = 'root';
$password = '';
$dbname = 'guestbook_db';

// ایجاد اتصال
$conn = new mysqli($host, $username, $password, $dbname);

// بررسی اتصال
if ($conn->connect_error) {
    die("❌ خطا در اتصال: " . $conn->connect_error);
}

// تنظیم charset برای پشتیبانی از فارسی
$conn->set_charset("utf8mb4");

echo "✅ اتصال با موفقیت برقرار شد.";

// بستن اتصال
$conn->close();
?>

۳. ایجاد فایل جداگانه برای اتصال (config/db.php):

<?php
// ===== فایل: config/db.php =====

// تنظیمات دیتابیس
define('DB_HOST', 'localhost');
define('DB_NAME', 'guestbook_db');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_CHARSET', 'utf8mb4');

/**
 * دریافت اتصال MySQLi (روش شیءگرا)
 * @return mysqli
 */
function getDBConnection() {
    $conn = new mysqli(DB_HOST, DB_USER, DB_PASS, DB_NAME);

    // بررسی اتصال
    if ($conn->connect_error) {
        error_log("Database Connection Error: " . $conn->connect_error);
        die("خطا در اتصال به دیتابیس. لطفاً با پشتیبانی تماس بگیرید.");
    }

    // تنظیم charset
    $conn->set_charset(DB_CHARSET);

    return $conn;
}

/**
 * پاک‌سازی ورودی برای نمایش امن
 * @param string $value
 * @return string
 */
function sanitize($value) {
    return htmlspecialchars(trim($value), ENT_QUOTES, 'UTF-8');
}

/**
 * بستن اتصال دیتابیس
 * @param mysqli $conn
 */
function closeDBConnection($conn) {
    if ($conn) {
        $conn->close();
    }
}
?>
📌 نکات مهم در MySQLi:
  • mysqli_connect_error() و $conn->connect_error برای نمایش خطای اتصال.
  • mysqli_set_charset() و $conn->set_charset() برای تنظیم charset.
  • mysqli_close() و $conn->close() برای بستن اتصال.
  • همیشه بعد از اتمام کار، اتصال را ببندید (مخصوصاً در پروژه‌های بزرگ).

۴. خطاهای رایج و عیب‌یابی:

خطا دلیل راه حل
Access denied for user نام کاربری یا رمز عبور اشتباه است نام کاربری و رمز را بررسی کنید
Unknown database دیتابیس وجود ندارد دیتابیس را در phpMyAdmin بسازید
Can't connect to MySQL server MySQL اجرا نمی‌شود XAMPP را باز کنید و MySQL را Start کنید
Call to undefined function mysqli_connect() پسوند mysqli نصب نیست در php.ini، extension=mysqli را فعال کنید

بخش چهارم: Prepared Statements با MySQLi (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «Prepared Statements یکی از مهم‌ترین ویژگی‌های MySQLi است. با این روش، کوئری و داده‌ها از هم جدا می‌شوند. فرآیند کار به این صورت است: ۱. کوئری را با placeholder (جای‌گذاری با علامت ?) آماده می‌کنیم. ۲. داده‌ها را با bind_param() به کوئری متصل می‌کنیم. ۳. کوئری را با execute() اجرا می‌کنیم. مزایا: - امنیت بالا (جلوگیری از SQL Injection) - سرعت بیشتر (کوئری یک بار کامپایل می‌شود) - خوانایی بهتر کد»

۱. ساختار Prepared Statements:

<?php
// ===== دریافت اتصال =====
$conn = getDBConnection();

// ===== ۱. آماده‌سازی کوئری =====
$stmt = $conn->prepare("INSERT INTO users (username, email, password) VALUES (?, ?, ?)");

// ===== ۲. اتصال داده‌ها به placeholderها =====
// s = string, i = integer, d = double, b = blob
$stmt->bind_param("sss", $username, $email, $password);

// ===== ۳. تنظیم مقادیر =====
$username = "ali_dev";
$email = "ali@site.com";
$password = "123456";

// ===== ۴. اجرا =====
if ($stmt->execute()) {
    echo "✅ کاربر با موفقیت ثبت شد.";
} else {
    echo "❌ خطا: " . $stmt->error;
}

// ===== ۵. بستن statement =====
$stmt->close();
$conn->close();
?>

۲. انواع داده‌ها در bind_param:

نوع داده نویسه توضیح مثال
رشته s String 'ali'
عدد صحیح i Integer 25
عدد اعشاری d Double 19.99
داده باینری b Blob تصاویر، فایل‌ها

۳. مثال‌های کامل با MySQLi:

<?php
require_once 'config/db.php';
$conn = getDBConnection();

// ===== SELECT با شرط =====
$stmt = $conn->prepare("SELECT * FROM users WHERE email = ? AND is_active = ?");
$stmt->bind_param("si", $email, $active);
$email = "ali@site.com";
$active = 1;
$stmt->execute();
$result = $stmt->get_result();
$user = $result->fetch_assoc();

// ===== SELECT با چندین رکورد =====
$stmt = $conn->prepare("SELECT * FROM users WHERE age > ? ORDER BY age DESC");
$stmt->bind_param("i", $min_age);
$min_age = 20;
$stmt->execute();
$result = $stmt->get_result();
$users = $result->fetch_all(MYSQLI_ASSOC);

// ===== INSERT =====
$stmt = $conn->prepare("INSERT INTO messages (user_id, title, message) VALUES (?, ?, ?)");
$stmt->bind_param("iss", $user_id, $title, $message);
$user_id = 1;
$title = "عنوان پیام";
$message = "متن پیام";
$stmt->execute();
$last_id = $stmt->insert_id;

// ===== UPDATE =====
$stmt = $conn->prepare("UPDATE messages SET is_approved = ? WHERE id = ?");
$stmt->bind_param("ii", $approved, $id);
$approved = 1;
$id = 5;
$stmt->execute();
$affected_rows = $stmt->affected_rows;

// ===== DELETE =====
$stmt = $conn->prepare("DELETE FROM messages WHERE id = ? AND user_id = ?");
$stmt->bind_param("ii", $id, $user_id);
$id = 10;
$user_id = 1;
$stmt->execute();

// ===== پیمایش با fetch =====
$stmt = $conn->prepare("SELECT * FROM messages WHERE user_id = ?");
$stmt->bind_param("i", $user_id);
$user_id = 1;
$stmt->execute();
$result = $stmt->get_result();

while ($row = $result->fetch_assoc()) {
    echo $row['title'] . " - " . $row['message'] . "<br>";
}

// ===== دریافت به‌صورت OBJECT =====
$stmt = $conn->prepare("SELECT * FROM users LIMIT ?");
$stmt->bind_param("i", $limit);
$limit = 5;
$stmt->execute();
$result = $stmt->get_result();

while ($user = $result->fetch_object()) {
    echo $user->username . " - " . $user->email . "<br>";
}

// ===== بستن اتصال =====
$stmt->close();
$conn->close();
?>
📌 نکات مهم در MySQLi Prepared Statements:
  • bind_param() برای اتصال داده‌ها به placeholderها استفاده می‌شود.
  • نوع داده‌ها را با حروف s، i، d، b مشخص کنید.
  • get_result() برای دریافت نتیجه از SELECT استفاده می‌شود.
  • fetch_assoc() → آرایه انجمنی، fetch_object() → شیء.
  • insert_id → شناسه آخرین رکورد درج شده.
  • affected_rows → تعداد رکوردهای تحت تأثیر قرار گرفته.
  • num_rows → تعداد رکوردهای برگشتی از SELECT.

بخش پنجم: تمرین عملی – ذخیره فرم در دیتابیس (۴۰ دقیقه)

۳. فایل save_message.php (ذخیره در دیتابیس با MySQLi):

<?php
// ===== فایل: save_message.php =====

// شامل فایل اتصال به دیتابیس
require_once 'config/db.php';

// بررسی اینکه فرم با POST ارسال شده است
if ($_SERVER["REQUEST_METHOD"] != "POST") {
    header('Location: index.php');
    exit;
}

// دریافت و پاک‌سازی داده‌ها
$username = isset($_POST['username']) ? trim($_POST['username']) : '';
$message = isset($_POST['message']) ? trim($_POST['message']) : '';

// ===== اعتبارسنجی =====
$errors = [];

if (empty($username)) {
    $errors[] = "نام کاربری الزامی است.";
} elseif (strlen($username) < 2 || strlen($username) > 50) {
    $errors[] = "نام کاربری باید بین ۲ تا ۵۰ کاراکتر باشد.";
}

// بررسی پیام
if (empty($message)) {
    $errors[] = "متن پیام الزامی است.";
} elseif (strlen($message) < 5) {
    $errors[] = "متن پیام باید حداقل ۵ کاراکتر باشد.";
}

// اگر خطایی وجود داشت، به فرم برگرد
if (!empty($errors)) {
    session_start();
    $_SESSION['errors'] = $errors;
    $_SESSION['old_data'] = [
        'username' => $username,
        'message' => $message
    ];
    header('Location: index.php');
    exit;
}

// ===== ذخیره در دیتابیس با MySQLi و Prepared Statement =====
$conn = getDBConnection();

// آماده‌سازی کوئری
$stmt = $conn->prepare("INSERT INTO messages (username, message, is_approved) VALUES (?, ?, 0)");

// اجرا
if ($stmt->execute()) {
    $stmt->close();
    $conn->close();
    header('Location: index.php?status=success');
    exit;
} else {
    // ثبت خطا در لاگ
    error_log("Error saving message: " . $stmt->error);
    $stmt->close();
    $conn->close();
    header('Location: index.php?status=error');

🧪 چالش‌های اضافی برای دانشجویان قوی‌تر:

چالش ۱: اعتبارسنجی پیشرفته

اعتبارسنجی زیر را اضافه کنید:

  • نام کاربری نباید شامل کاراکترهای خاص باشد (فقط حروف و اعداد).
  • پیام نباید شامل لینک‌های مخرب باشد.
  • محدودیت تعداد پیام‌های هر کاربر در روز (مثلاً ۵ پیام).

چالش ۲: نمایش پیام‌ها در صفحه اصلی

در index.php، آخرین ۵ پیام تأیید شده را از دیتابیس بخوانید و نمایش دهید.

راهنما: از SELECT ... WHERE is_approved = 1 ORDER BY created_at DESC LIMIT 5

چالش ۳: تابع ذخیره‌سازی

یک تابع به نام saveMessage($username, $message) در db.php بسازید و از آن در save_message.php استفاده کنید.

چالش ۴: جلوگیری از ارسال مجدد فرم

بعد از ارسال موفق، کاربر را با header('Location: ...') هدایت کنید تا با رفرش صفحه، فرم دوباره ارسال نشود.
