جلسه ۰۸: پیاده‌سازی عملیات ایجاد و ویرایش (Create & Update)

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: تکمیل عملیات CRUD با افزودن Create و Update

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

در این جلسه، دو عملیات باقی‌مانده از CRUD را کامل می‌کنیم: ایجاد (Create) و ویرایش (Update). با این جلسه، دانشجویان توانایی انجام تمام عملیات‌های اصلی روی داده‌ها را خواهند داشت.

  1. مرور CRUD و ایجاد (Create) (۲۰ دقیقه): مرور جلسات قبل، تکمیل فرم عمومی برای افزودن پیام.
  2. ویرایش (Update) – قسمت اول (۳۰ دقیقه): خواندن داده‌های یک رکورد خاص برای فرم ویرایش.
  3. ویرایش – قسمت دوم (۳۰ دقیقه): ذخیره تغییرات با UPDATE و Prepared Statement.
  4. تمرین عملی: فرم ویرایش پیام (۴۰ دقیقه): ساخت صفحه ویرایش کامل با اعتبارسنجی و به‌روزرسانی.
📌 نکته مهم برای مدرس:
  • این جلسه، CRUD را کامل می‌کند. دانشجویان باید تمام عملیات را درک کنند.
  • تأکید کنید که UPDATE هم مانند DELETE نیاز به WHERE دارد.
  • نحوه بارگذاری داده‌ها در فرم ویرایش را با دقت توضیح دهید.
  • از header() برای هدایت کاربر بعد از عملیات استفاده کنید.

بخش دوم: مرور CRUD و ایجاد (Create) (۲۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان با سه عملیات CRUD آشنا شدیم: - Create (ایجاد) → INSERT (جلسه ۶) - Read (خواندن) → SELECT (جلسه ۷) - Delete (حذف) → DELETE (جلسه ۷) در این جلسه، Update (به‌روزرسانی) را کامل می‌کنیم. با این جلسه، شما یک سیستم CRUD کامل خواهید داشت!»

نمودار چرخه CRUD:

📝 Create → 📖 Read → ✏️ Update → 🗑️ Delete

مرور Create با MySQLi:

<?php
// ===== فایل: save_message.php (MySQLi) =====
require_once 'config/db.php';

if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $username = trim($_POST['username']);
    $message = trim($_POST['message']);
    
    $errors = [];
    if (empty($username)) $errors[] = "نام کاربری الزامی است.";
    if (empty($message)) $errors[] = "متن پیام الزامی است.";
    
    if (empty($errors)) {
        $conn = getDBConnection();
        $stmt = $conn->prepare("INSERT INTO messages (username, message, is_approved) VALUES (?, ?, 0)");
        $stmt->bind_param("ss", $username, $message);
        $stmt->execute();
        $stmt->close();
        $conn->close();
        header('Location: index.php?status=success');
        exit;
    }
}
?>

بخش سوم: ویرایش (Update) – خواندن داده برای فرم (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «برای ویرایش یک رکورد، باید دو مرحله را انجام دهیم: مرحله ۱: داده‌های رکورد را از دیتابیس بخوانیم و در فرم نمایش دهیم. مرحله ۲: تغییرات را دریافت کنیم و با UPDATE در دیتابیس ذخیره کنیم. در این بخش، مرحله ۱ را انجام می‌دهیم. شناسه رکورد را از Query String دریافت می‌کنیم و با SELECT داده‌ها را می‌خوانیم. نکته مهم: اگر رکورد وجود نداشت، کاربر را به صفحه مدیریت بازگردانید.»

۱. خواندن داده برای ویرایش با MySQLi:

<?php
// ===== فایل: edit.php (MySQLi) =====
require_once 'config/db.php';

if (!isset($_GET['id'])) {
    header('Location: admin.php?error=no_id');
    exit;
}

$id = (int)$_GET['id'];

if ($id <= 0) {
    header('Location: admin.php?error=invalid_id');
    exit;
}

$conn = getDBConnection();

$stmt = $conn->prepare("SELECT id, username, message, is_approved, created_at FROM messages WHERE id = ?");
$stmt->bind_param("i", $id);
$stmt->execute();
$result = $stmt->get_result();
$message = $result->fetch_assoc();
$stmt->close();

if (!$message) {
    $conn->close();
    header('Location: admin.php?error=not_found');
    exit;
}

$conn->close();
?>

بخش چهارم: ویرایش – ذخیره تغییرات با UPDATE (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «حالا به مرحله دوم می‌رسیم: ذخیره تغییرات در دیتابیس. برای به‌روزرسانی از دستور UPDATE استفاده می‌کنیم. مهم‌ترین نکته: همیشه از WHERE استفاده کنید! اگر WHERE را فراموش کنید، همه رکوردها تغییر می‌کنند. همچنین باید: ۱. داده‌ها را اعتبارسنجی کنید. ۲. از Prepared Statements استفاده کنید. ۳. کاربر را بعد از عملیات هدایت کنید.»

۱. ساختار UPDATE با MySQLi:

<?php
// ===== فایل: update.php (MySQLi) =====
require_once 'config/db.php';

if ($_SERVER["REQUEST_METHOD"] != "POST") {
    header('Location: admin.php');
    exit;
}

$id = isset($_POST['id']) ? (int)$_POST['id'] : 0;
$username = isset($_POST['username']) ? trim($_POST['username']) : '';
$message = isset($_POST['message']) ? trim($_POST['message']) : '';
$is_approved = isset($_POST['is_approved']) ? (int)$_POST['is_approved'] : 0;

$errors = [];

if ($id <= 0) {
    $errors[] = "شناسه نامعتبر است.";
}

if (empty($username)) {
    $errors[] = "نام کاربری الزامی است.";
} elseif (strlen($username) < 2 || strlen($username) > 50) {
    $errors[] = "نام کاربری باید بین ۲ تا ۵۰ کاراکتر باشد.";
}

if (empty($message)) {
    $errors[] = "متن پیام الزامی است.";
} elseif (strlen($message) < 5) {
    $errors[] = "متن پیام باید حداقل ۵ کاراکتر باشد.";
}

if (!empty($errors)) {
    session_start();
    $_SESSION['update_errors'] = $errors;
    $_SESSION['update_data'] = [
        'id' => $id,
        'username' => $username,
        'message' => $message,
        'is_approved' => $is_approved
    ];
    header('Location: edit.php?id=' . $id);
    exit;
}

$conn = getDBConnection();

$check_stmt = $conn->prepare("SELECT id FROM messages WHERE id = ?");
$check_stmt->bind_param("i", $id);
$check_stmt->execute();
$check_result = $check_stmt->get_result();

if ($check_result->num_rows == 0) {
    $check_stmt->close();
    $conn->close();
    header('Location: admin.php?error=not_found');
    exit;
}
$check_stmt->close();

$update_stmt = $conn->prepare("
    UPDATE messages 
    SET username = ?, 
        message = ?, 
        is_approved = ? 
    WHERE id = ?
");
$update_stmt->bind_param("ssii", $username, $message, $is_approved, $id);

if ($update_stmt->execute()) {
    $update_stmt->close();
    $conn->close();
    header('Location: admin.php?success=updated');
} else {
    $update_stmt->close();
    $conn->close();
    header('Location: admin.php?error=update_failed');
}
exit;
?>

۲. UPDATE با چندین شرط:

<?php
// ===== به‌روزرسانی با چند شرط =====
$stmt = $conn->prepare("
    UPDATE messages 
    SET is_approved = ? 
    WHERE user_id = ? AND created_at > ?
");
$stmt->bind_param("iis", $approved, $user_id, $date);
$approved = 1;
$user_id = 5;
$date = '2024-01-01';
$stmt->execute();

// ===== به‌روزرسانی با محاسبه =====
$stmt = $conn->prepare("UPDATE users SET age = age + 1 WHERE is_active = ?");
$stmt->bind_param("i", $active);
$active = 1;
$stmt->execute();
?>
⚠️ هشدار مهم:
  • همیشه از WHERE در UPDATE استفاده کنید.
  • اگر WHERE را فراموش کنید، همه رکوردها تغییر می‌کنند!
  • از affected_rows برای بررسی اینکه آیا رکوردی تغییر کرده است استفاده کنید.
