جلسه ۰۷: پیاده‌سازی عملیات خواندن و حذف (Read & Delete)

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: نمایش داده‌ها از دیتابیس و حذف امن با Query String

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

در این جلسه، دانشجویان با دو عملیات اصلی CRUD آشنا می‌شوند: خواندن (Read) و حذف (Delete). این عملیات پایه‌ای‌ترین عملیات‌های هر سیستم مدیریت محتوا هستند.

  1. مرور MySQLi و SELECT (۲۰ دقیقه): مرور مفاهیم جلسه قبل، دریافت و نمایش داده‌ها با MySQLi.
  2. Query String و پارامترهای URL (۳۰ دقیقه): نحوه انتقال داده از طریق URL، دریافت با $_GET، کاربردها.
  3. پیاده‌سازی حذف با DELETE (۳۰ دقیقه): نوشتن کوئری DELETE با Prepared Statement، تایید حذف.
  4. تمرین عملی: نمایش و حذف پیام‌ها (۴۰ دقیقه): ساخت صفحه مدیریت پیام‌ها با قابلیت نمایش و حذف.
📌 نکته مهم برای مدرس:
  • تأکید کنید که حذف یک عملیات حساس است و باید با دقت انجام شود.
  • حتماً مفهوم Query String را با مثال‌های عملی نشان دهید.
  • استفاده از تأیید حذف (با جاوااسکریپت یا صفحه جداگانه) را توضیح دهید.
  • تأکید کنید که همیشه از WHERE در DELETE استفاده شود.

بخش دوم: خواندن داده‌ها با MySQLi (۲۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «در جلسه قبل یاد گرفتیم که چطور داده‌ها را در دیتابیس ذخیره کنیم. حالا نوبت به خواندن داده‌هاست. در یک سیستم مدیریت محتوا، ما دائماً نیاز داریم که داده‌ها را از دیتابیس بخوانیم و به کاربر نمایش دهیم. مثلاً لیست محصولات، لیست کاربران، یا لیست پیام‌ها. برای خواندن داده‌ها از SELECT استفاده می‌کنیم. با MySQLi، می‌توانیم داده‌ها را به‌صورت آرایه یا شیء دریافت کنیم.»

۱. خواندن همه رکوردها:

<?php
require_once 'config/db.php';
$conn = getDBConnection();

// ===== خواندن همه پیام‌ها =====
$result = $conn->query("SELECT * FROM messages ORDER BY created_at DESC");

// ===== نمایش داده‌ها =====
while ($message = $result->fetch_assoc()) {
    echo "نام: " . $message['username'] . "<br>";
    echo "پیام: " . $message['message'] . "<br>";
    echo "تاریخ: " . $message['created_at'] . "<br><br>";
}

// ===== تعداد رکوردها =====
$count = $result->num_rows;
echo "تعداد کل پیام‌ها: " . $count;

$conn->close();
?>

۲. خواندن با شرط و Prepared Statement:

<?php
require_once 'config/db.php';
$conn = getDBConnection();

// ===== خواندن پیام‌های تأیید شده =====
$stmt = $conn->prepare("SELECT * FROM messages WHERE is_approved = ?");
$stmt->bind_param("i", $approved);
$approved = 1;
$stmt->execute();
$result = $stmt->get_result();
$approved_messages = $result->fetch_all(MYSQLI_ASSOC);

// ===== خواندن با محدودیت (صفحه‌بندی) =====
$page = 1;
$per_page = 10;
$offset = ($page - 1) * $per_page;

$stmt = $conn->prepare("SELECT * FROM messages ORDER BY created_at DESC LIMIT ? OFFSET ?");
$stmt->bind_param("ii", $per_page, $offset);
$stmt->execute();
$result = $stmt->get_result();
$paginated_messages = $result->fetch_all(MYSQLI_ASSOC);

// ===== تعداد کل رکوردها =====
$count_result = $conn->query("SELECT COUNT(*) as total FROM messages");
$total = $count_result->fetch_assoc()['total'];
$total_pages = ceil($total / $per_page);

$conn->close();
?>

۳. روش‌های مختلف fetch:

متد توضیح نحوه دسترسی
fetch_assoc() آرایه انجمنی $row['column_name']
fetch_object() شیء $row->column_name
fetch_row() آرایه عددی $row[0]
fetch_all(MYSQLI_ASSOC) همه رکوردها به‌صورت آرایه $rows[0]['column_name'] 
<?php
require_once 'config/db.php';
$conn = getDBConnection();

// ===== مثال fetch_object() =====
$result = $conn->query("SELECT id, username, message FROM messages LIMIT 5");
while ($row = $result->fetch_object()) {
    echo $row->username . ": " . $row->message . "<br>";
}

// ===== مثال fetch_all() =====
$result = $conn->query("SELECT username FROM users");
$usernames = $result->fetch_all(MYSQLI_ASSOC);
foreach ($usernames as $user) {
    echo $user['username'] . "<br>";
}

$conn->close();
?>

بخش سوم: Query String و پارامترهای URL (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «Query String بخشی از URL است که بعد از علامت ? قرار می‌گیرد. از آن برای انتقال داده‌های کوچک بین صفحات استفاده می‌شود. کاربردهای Query String: - انتقال شناسه رکورد برای ویرایش یا حذف - صفحه‌بندی (Pagination) - فیلتر کردن داده‌ها - جستجو - مرتب‌سازی در PHP، داده‌های Query String را با $_GET دریافت می‌کنیم.»

۱. ساختار و کاربرد Query String:

<?php
// ===== دریافت پارامترها از URL =====
// URL: http://localhost/admin.php?page=3&sort=date&filter=approved

$page = isset($_GET['page']) ? (int)$_GET['page'] : 1;
$sort = isset($_GET['sort']) ? $_GET['sort'] : 'date';
$filter = isset($_GET['filter']) ? $_GET['filter'] : 'all';

echo "صفحه: $page, مرتب‌سازی: $sort, فیلتر: $filter";

// ===== ساخت لینک با پارامترهای GET =====
echo '<a href="?page=2&sort=name">صفحه ۲ با مرتب‌سازی نام</a>';

// ===== ساخت لینک حذف =====
echo '<a href="delete.php?id=' . $message['id'] . '" onclick="return confirm(\'آیا مطمئن هستید؟\')">حذف</a>';
?>

۲. مثال کامل: صفحه‌بندی با Query String:

<?php
require_once 'config/db.php';
$conn = getDBConnection();

// ===== تنظیمات صفحه‌بندی =====
$per_page = 5;
$page = isset($_GET['page']) ? (int)$_GET['page'] : 1;
if ($page < 1) $page = 1;
$offset = ($page - 1) * $per_page;

// ===== دریافت داده‌ها =====
$stmt = $conn->prepare("SELECT * FROM messages ORDER BY created_at DESC LIMIT ? OFFSET ?");
$stmt->bind_param("ii", $per_page, $offset);
$stmt->execute();
$result = $stmt->get_result();
$messages = $result->fetch_all(MYSQLI_ASSOC);

// ===== تعداد کل رکوردها =====
$count_result = $conn->query("SELECT COUNT(*) as total FROM messages");
$total = $count_result->fetch_assoc()['total'];
$total_pages = ceil($total / $per_page);

// ===== نمایش لینک‌های صفحه‌بندی =====
echo '<div class="pagination">';

// لینک صفحه قبلی
if ($page > 1) {
    echo '<a href="?page=' . ($page - 1) . '">قبلی</a>';
}

// شماره صفحات
for ($i = 1; $i <= $total_pages; $i++) {
    if ($i == $page) {
        echo '<strong>' . $i . '</strong>';
    } else {
        echo '<a href="?page=' . $i . '">' . $i . '</a>';
    }
}

// لینک صفحه بعدی
if ($page < $total_pages) {
    echo '<a href="?page=' . ($page + 1) . '">بعدی</a>';
}

echo '</div>';

$conn->close();
?>

بخش چهارم: پیاده‌سازی حذف با DELETE (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «عملیات حذف یکی از حساس‌ترین عملیات‌های هر سیستم است. اگر اشتباه انجام شود، داده‌های مهمی را از دست می‌دهیم. برای حذف از دستور DELETE استفاده می‌کنیم. اما قبل از حذف، باید: ۱. مطمئن شویم که کاربر مجوز حذف دارد. ۲. از کاربر تأیید بگیریم. ۳. شناسه رکورد را به‌درستی دریافت کنیم. ۴. از Prepared Statement برای امنیت استفاده کنیم.»

۱. ساختار پایه حذف:

<?php
// ===== فایل: delete.php =====
require_once 'config/db.php';

// ===== دریافت شناسه =====
if (!isset($_GET['id'])) {
    header('Location: admin.php?error=no_id');
    exit;
}

$id = (int)$_GET['id'];

// ===== حذف با Prepared Statement =====
$conn = getDBConnection();

$stmt = $conn->prepare("DELETE FROM messages WHERE id = ?");
$stmt->bind_param("i", $id);

if ($stmt->execute()) {
    // بررسی اینکه آیا رکوردی حذف شده است
    if ($stmt->affected_rows > 0) {
        header('Location: admin.php?success=deleted');
    } else {
        header('Location: admin.php?error=not_found');
    }
} else {
    header('Location: admin.php?error=delete_failed');
}

$stmt->close();
$conn->close();
exit;
?>

۲. حذف با تأیید جاوااسکریپت:

<?php
// ===== در صفحه admin.php =====
// لینک حذف با تأیید
echo '<a href="delete.php?id=' . $message['id'] . '" 
        onclick="return confirm(\'آیا از حذف این پیام مطمئن هستید؟\')" 
        class="delete-btn">
    🗑️ حذف
</a>';
?>

۳. حذف با متد POST (امن‌تر):

<?php
// ===== فرم حذف با POST =====
// در admin.php
echo '<form method="POST" action="delete.php" style="display:inline;" 
        onsubmit="return confirm(\'آیا از حذف این پیام مطمئن هستید؟\')">
    <input type="hidden" name="id" value="' . $message['id'] . '">
    <button type="submit" class="delete-btn">🗑️ حذف</button>
</form>';

// ===== فایل delete.php با POST =====
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    $id = isset($_POST['id']) ? (int)$_POST['id'] : 0;
    
    if ($id > 0) {
        $conn = getDBConnection();
        $stmt = $conn->prepare("DELETE FROM messages WHERE id = ?");
        $stmt->bind_param("i", $id);
        $stmt->execute();
        
        if ($stmt->affected_rows > 0) {
            header('Location: admin.php?success=deleted');
        } else {
            header('Location: admin.php?error=not_found');
        }
        $stmt->close();
        $conn->close();
        exit;
    }
}
?>
⚠️ هشدار مهم:
  • همیشه از WHERE در DELETE استفاده کنید.
  • از affected_rows برای بررسی اینکه آیا رکوردی حذف شده است استفاده کنید.
  • برای حذف چندین رکورد، از IN با آرایه استفاده کنید.

بخش پنجم: تمرین عملی – نمایش و حذف پیام‌ها (۴۰ دقیقه)

🗣 مأموریت کلاسی: «حالا یک صفحه مدیریت کامل برای Guestbook می‌سازیم. این صفحه تمام پیام‌های ثبت شده را نمایش می‌دهد و امکان حذف آن‌ها را فراهم می‌کند. ویژگی‌های صفحه مدیریت: - نمایش تمام پیام‌ها با جزئیات کامل - نمایش وضعیت تأیید هر پیام - دکمه حذف با تأیید جاوااسکریپت - نمایش تعداد کل پیام‌ها - صفحه‌بندی (Pagination) - پیام‌های موفقیت و خطا»

فایل delete.php (پردازش حذف با MySQLi):

<?php
// ===== فایل: delete.php (نسخه MySQLi) =====
require_once 'config/db.php';

// ===== بررسی روش ارسال =====
if ($_SERVER["REQUEST_METHOD"] != "POST") {
    header('Location: admin.php?error=invalid_method');
    exit;
}

// ===== دریافت شناسه =====
if (!isset($_POST['id'])) {
    header('Location: admin.php?error=no_id');
    exit;
}

$id = (int)$_POST['id'];

if ($id <= 0) {
    header('Location: admin.php?error=invalid_id');
    exit;
}

// ===== حذف از دیتابیس =====
$conn = getDBConnection();

// ابتدا بررسی کنیم که پیام وجود دارد
$check_stmt = $conn->prepare("SELECT id FROM messages WHERE id = ?");
$check_stmt->bind_param("i", $id);
$check_stmt->execute();
$check_result = $check_stmt->get_result();

if ($check_result->num_rows == 0) {
    $check_stmt->close();
    $conn->close();
    header('Location: admin.php?error=not_found');
    exit;
}
$check_stmt->close();

// حذف پیام
$delete_stmt = $conn->prepare("DELETE FROM messages WHERE id = ?");
$delete_stmt->bind_param("i", $id);
$delete_stmt->execute();

if ($delete_stmt->affected_rows > 0) {
    $delete_stmt->close();
    $conn->close();
    header('Location: admin.php?success=deleted');
} else {
    $delete_stmt->close();
    $conn->close();
    header('Location: admin.php?error=delete_failed');
}
exit;
?>
📌 خروجی نهایی قابل قبول:
  • صفحه admin.php تمام پیام‌ها را با صفحه‌بندی نمایش می‌دهد.
  • دکمه حذف با تأیید جاوااسکریپت کار می‌کند.
  • پس از حذف، صفحه رفرش می‌شود و پیام موفقیت نمایش داده می‌شود.
  • خطاها به‌درستی مدیریت و نمایش داده می‌شوند.
