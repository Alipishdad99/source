جلسه ۰۵: طراحی پایگاه داده (MySQL)

مدت زمان تدریس: ۱۲۰ دقیقه | هدف: آشنایی با مفاهیم RDBMS، طراحی جدول و نوشتن کوئری‌های پایه SQL

بخش اول: زمان‌بندی جلسه (۱۲۰ دقیقه)

این جلسه، دانشجویان را با دنیای پایگاه داده آشنا می‌کند. مفاهیمی که در این جلسه یاد گرفته می‌شود، پایه و اساس تمام پروژه‌های دیتابیس‌محور است.

  1. معرفی پایگاه داده رابطه‌ای (۲۰ دقیقه): مفاهیم جدول، رکورد، فیلد، کلید اصلی، روابط و مدل‌سازی داده.
  2. آشنایی با phpMyAdmin (۳۰ دقیقه): محیط phpMyAdmin، ایجاد دیتابیس، ایجاد جدول، درک ساختار.
  3. دستورات SQL پایه (۳۰ دقیقه): CREATE TABLE، INSERT INTO، SELECT، WHERE، ORDER BY.
  4. تمرین عملی: طراحی جدول کاربران و پیام‌ها (۴۰ دقیقه): ایجاد دو جدول و درج داده با کوئری‌های SQL.
📌 نکته مهم برای مدرس:
  • این جلسه، پایه‌ای‌ترین جلسه برای درک دیتابیس است.
  • از مثال‌های روزمره برای توضیح مفاهیم استفاده کنید (دفترچه تلفن، لیست خرید، سیستم نمرات).
  • حتماً دانشجویان را به phpMyAdmin ببرید و عملاً جدول بسازید.
  • تأکید کنید که کلید اصلی (Primary Key) مهم‌ترین مفهوم در طراحی جدول است.
  • تفاوت بین CHAR و VARCHAR و TEXT را توضیح دهید.

بخش دوم: مفاهیم پایگاه داده رابطه‌ای (۲۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «تا الان، داده‌هایمان را در حافظه موقت (متغیرها و آرایه‌ها) ذخیره می‌کردیم. اما وقتی برنامه را می‌بندیم، همه داده‌ها از بین می‌روند! برای اینکه داده‌ها را دائمی نگهداری کنیم، به یک پایگاه داده نیاز داریم. پایگاه داده مثل یک دفترچه بزرگ است که اطلاعات را به‌صورت منظم و ساختاریافته ذخیره می‌کند. در پایگاه داده رابطه‌ای، اطلاعات در جدول‌ها ذخیره می‌شوند. به یک جدول مثل یک صفحه از دفترچه نگاه کنید: - هر سطر (Row) یک رکورد (Record) است (یک واحد کامل اطلاعات). - هر ستون (Column) یک فیلد (Field) است (یک ویژگی از اطلاعات). برای مثال، جدول کاربران:
┌────┬──────────┬─────────────────┬──────────┐
│ id │username │ email │ password │
├────┼──────────┼─────────────────┼──────────┤
│ 1 │ ali │ali@site.com │ 123456 │
│ 2 │ sara │ sara@site.com │ 654321 │
└────┴──────────┴─────────────────┴──────────┘

هر سطر یک کاربر است و هر ستون یک ویژگی از آن کاربر.»

۱. مفاهیم کلیدی:

  • جدول (Table): مجموعه‌ای از رکوردها با ساختار یکسان. مثل یک صفحه در دفترچه که همه سطرها دارای ستون‌های یکسانی هستند.
  • رکورد (Record / Row): یک واحد کامل از اطلاعات. مثل اطلاعات یک کاربر خاص.
  • فیلد (Field / Column): یک ویژگی از اطلاعات. مثل نام، ایمیل یا رمز عبور.
  • کلید اصلی (Primary Key): یک فیلد منحصر‌به‌فرد که هر رکورد را به‌طور یکتا شناسایی می‌کند. مثل کد ملی یا شماره شناسنامه. در جدول بالا، id کلید اصلی است.
  • کلید خارجی (Foreign Key): فیلدی که به کلید اصلی یک جدول دیگر اشاره می‌کند. برای ایجاد ارتباط بین جدول‌ها استفاده می‌شود.

۲. انواع داده‌ها در MySQL:

نوع داده توضیح مثال
INT اعداد صحیح age INT
VARCHAR(n) رشته با طول متغیر (حداکثر n کاراکتر) name VARCHAR(50)
TEXT متن طولانی (حداکثر ۶۵۵۳۵ کاراکتر) description TEXT
DATE تاریخ (YYYY-MM-DD) birthdate DATE
DATETIME تاریخ و زمان created_at DATETIME
DECIMAL(p,s) اعداد اعشاری دقیق price DECIMAL(10,2)
BOOLEAN مقدار درست/نادرست (۰ یا ۱) is_active BOOLEAN

📌 رابطه بین جدول‌ها:

هر کاربر می‌تواند چندین پیام داشته باشد (رابطه یک‌به‌چند).

user_id در جدول messages به id در جدول users اشاره می‌کند.

این یعنی هر پیام به یک کاربر خاص تعلق دارد.

بخش چهارم: دستورات SQL پایه (۳۰ دقیقه)

🗣 کلمه به کلمه چه بگویید: «SQL مخفف Structured Query Language است. زبان استاندارد برای کار با پایگاه داده‌های رابطه‌ای. چهار دستور اصلی که با آن‌ها کار می‌کنیم (CRUD): - Create → INSERT (درج داده) - Read → SELECT (خواندن داده) - Update → UPDATE (به‌روزرسانی داده) - Delete → DELETE (حذف داده)»

۱. CREATE TABLE (ایجاد جدول):

-- ===== جدول کاربران =====
CREATE TABLE users (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    age INT(3) DEFAULT '0',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- ===== جدول پیام‌ها =====
CREATE TABLE messages (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    user_id INT(11) NOT NULL,
    message TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
📌 توضیح کلمات کلیدی:
  • AUTO_INCREMENT: مقدار به‌طور خودکار افزایش می‌یابد.
  • PRIMARY KEY: کلید اصلی (منحصر‌به‌فرد و غیرتکراری).
  • NOT NULL: فیلد نمی‌تواند خالی باشد.
  • UNIQUE: مقدار فیلد باید منحصر‌به‌فرد باشد.
  • DEFAULT: مقدار پیش‌فرض در صورت خالی بودن.
  • CURRENT_TIMESTAMP: زمان فعلی را ذخیره می‌کند.
  • FOREIGN KEY: کلید خارجی برای ارتباط با جدول دیگر.
  • ON DELETE CASCADE: اگر کاربر حذف شود، پیام‌هایش هم حذف شوند.

۳. SELECT (خواندن داده):

-- ===== خواندن همه کاربران =====
SELECT * FROM users;

-- ===== خواندن فقط فیلدهای خاص =====
SELECT id, username, email FROM users;

-- ===== خواندن با شرط =====
SELECT * FROM users WHERE age > 25;

-- ===== خواندن با شرط دقیق =====
SELECT * FROM users WHERE username = 'ali_r';

-- ===== خواندن با مرتب‌سازی =====
SELECT * FROM users ORDER BY created_at DESC;  -- جدیدترین اول
SELECT * FROM users ORDER BY age ASC;       -- کوچک‌ترین سن اول

-- ===== خواندن با محدودیت =====
SELECT * FROM users LIMIT 5;           -- فقط ۵ کاربر اول
SELECT * FROM users LIMIT 10 OFFSET 5;  -- ۱۰ کاربر از ردیف ۶

-- ===== خواندن با COUNT (تعداد رکوردها) =====
SELECT COUNT(*) AS total_users FROM users;

-- ===== خواندن با JOIN (اتصال دو جدول) =====
SELECT
    u.username,
    u.email,
    m.message,
    m.created_at AS message_date
FROM users u
JOIN messages m ON u.id = m.user_id
ORDER BY m.created_at DESC;
📌 نکات مهم در SELECT:
  • * یعنی همه فیلدها. برای خوانایی بهتر، فقط فیلدهای مورد نیاز را بنویسید.
  • AS برای تغییر نام ستون در خروجی استفاده می‌شود.
  • JOIN برای ترکیب داده‌های دو یا چند جدول استفاده می‌شود.
  • WHERE برای فیلتر کردن رکوردها.
  • ORDER BY برای مرتب‌سازی (ASC صعودی، DESC نزولی).
  • LIMIT برای محدود کردن تعداد رکوردها.

۴. UPDATE (به‌روزرسانی داده):

-- ===== به‌روزرسانی یک کاربر =====
UPDATE users
SET email = 'ali_new@site.com', age = 26
WHERE id = 1;

-- ===== به‌روزرسانی چند کاربر با شرط =====
UPDATE users
SET age = age + 1
WHERE age < 20;

-- ⚠️ خطرناک: بدون WHERE همه رکوردها تغییر می‌کنند!
UPDATE users SET password = 'newpass';  -- همه کاربران

۵. DELETE (حذف داده):

-- ===== حذف یک کاربر خاص =====
DELETE FROM users WHERE id = 4;

-- ===== حذف با شرط =====
DELETE FROM users WHERE age < 18;

-- ===== حذف همه رکوردها (محتاط باشید!) =====
DELETE FROM users;  -- همه کاربران حذف می‌شوند

-- ===== خالی کردن جدول (بهتر از DELETE) =====
TRUNCATE TABLE users;  -- جدول را خالی می‌کند و AUTO_INCREMENT را ریست می‌کند
⚠️ هشدار مهم:
  • همیشه قبل از UPDATE و DELETE، از WHERE استفاده کنید.
  • اگر WHERE را فراموش کنید، همه رکوردها تغییر/حذف می‌شوند!
  • برای تست، اول با SELECT بررسی کنید که شرط درست است.

بخش پنجم: تمرین عملی – طراحی جدول کاربران و پیام‌ها (۴۰ دقیقه)

🗣 مأموریت کلاسی: «حالا نوبت شماست که دست به کار شوید. یک دیتابیس کامل برای پروژه Guestbook طراحی کنید. مراحل: ۱. وارد phpMyAdmin شوید. ۲. یک دیتابیس جدید به نام guestbook_db بسازید. ۳. دو جدول users و messages ایجاد کنید. ۴. چند کاربر و پیام نمونه درج کنید. ۵. کوئری‌های مختلف SELECT بنویسید و نتیجه را ببینید.»

کوئری‌های کامل برای اجرا در phpMyAdmin:

-- ============================================
-- ۱. ایجاد دیتابیس
-- ============================================
CREATE DATABASE IF NOT EXISTS guestbook_db
CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;

USE guestbook_db;

-- ============================================
-- ۲. ایجاد جدول کاربران
-- ============================================
CREATE TABLE IF NOT EXISTS users (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(255) NOT NULL,
    full_name VARCHAR(100) DEFAULT NULL,
    age INT(3) DEFAULT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- ============================================
-- ۳. ایجاد جدول پیام‌ها
-- ============================================
CREATE TABLE IF NOT EXISTS messages (
    id INT(11) AUTO_INCREMENT PRIMARY KEY,
    user_id INT(11) NOT NULL,
    title VARCHAR(200) DEFAULT NULL,
    message TEXT NOT NULL,
    is_approved BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);

-- ============================================
-- ۶. کوئری‌های تست
-- ============================================

-- ۶-۱. نمایش همه کاربران
SELECT * FROM users;

-- ۶-۲. نمایش کاربران بالای ۲۰ سال
SELECT id, username, full_name, age
FROM users
WHERE age > 20
ORDER BY age DESC;

-- ۶-۳. نمایش پیام‌های تأیید شده
SELECT * FROM messages
WHERE is_approved = TRUE
ORDER BY created_at DESC;

-- ۶-۴. نمایش پیام‌های یک کاربر خاص با JOIN
SELECT
    u.username,
    u.full_name,
    m.title,
    m.message,
    m.created_at AS sent_at,
    CASE WHEN m.is_approved = 1 THEN '✅ تأیید شده'
        ELSE '⏳ در انتظار تأیید'
    END AS status
FROM users u
JOIN messages m ON u.id = m.user_id
WHERE u.username = 'ali_developer'
ORDER BY m.created_at DESC;

-- ۶-۵. آمار تعداد پیام‌ها به ازای هر کاربر
SELECT
    u.username,
    COUNT(m.id) AS message_count,
    SUM(CASE WHEN m.is_approved = 1 THEN 1 ELSE 0 END) AS approved_count
FROM users u
LEFT JOIN messages m ON u.id = m.user_id
GROUP BY u.id
ORDER BY message_count DESC;

-- ۶-۶. جدیدترین ۵ پیام
SELECT
    u.username,
    m.title,
    SUBSTRING(m.message, 1, 50) AS preview,
    m.created_at
FROM messages m
JOIN users u ON m.user_id = u.id
WHERE m.is_approved = TRUE
ORDER BY m.created_at DESC
LIMIT 5;

  2. روی «New» کلیک کنید و دیتابیس guestbook_db را بسازید.
  3. وارد دیتابیس شوید و روی تب «SQL» کلیک کنید.
  4. کوئری‌های CREATE TABLE را کپی کنید و اجرا کنید.
  5. کوئری‌های INSERT را کپی کنید و اجرا کنید.
  6. کوئری‌های SELECT مختلف را تست کنید و نتایج را بررسی کنید.
🧪 چالش‌های اضافی برای دانشجویان قوی‌تر:

چالش ۲: کوئری با GROUP BY

چالش ۳: کوئری با تاریخ
